Network Open System Interconnection (OSI) Model
Layer Protocol E.g. PDU What the protocol is controlling for Operators
1. Physical USB, RS-232, Ethernet ... bit raw bit transmission through transmission medium (i.e. cable) Cable, hubs, repeaters
2. Data Link MAC, LLC ... frame data frames transmission between two node connected by physical layer Layer 2 switch
3. Network IPv4, IPv6 ... packet multi-node network structure
(routing/traffic control/addressing)
4. Transport TCP, UDP ... segment transmission of data segments from source host to destination host across network
(error recovery / segmentation and reassembly / ...)
5. Session NetBIOS ... data
6. Presentation ASCII, MIME ... (character encoding / encryption and decryption / ...)
7. Application FTP, HTTP, SSH ... API

Next Hop vs Default Gateway
Default gateway  →  through which a device can access other network
Next hop  →  next router(/device) in the path to a specific destination
E.g.
--------------------------------------
Device A IP: 192.168.4.10
 Default Gateway: 192.168.4.1
Device B IP: 182.168.4.21
 Default Gateway: (blank)
Device C IP: 182.168.12.9

Subnet Mask: 255.255.255.0
 i.e. Device C is in different network
--------------------------------------
Within Network:
when A trying to send packets to B,
A applies Subnet Mask to IP-A and IP-B to deduce the Network ID → same = same network
resolve IP into MAC with ARP

Outside Network:
when A trying to send packets to C,
from AND operation with Subnet Mask A knows C is not on the same network,
so it sets the next-hop MAC address to gateway device

B doesn't have a default gateway set → B cannot access the Internet

  • routers uses its subnet mask to deduce the host ID
  • send data to a subnet
